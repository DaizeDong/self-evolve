# self-evolve 的哲学

## Thesis

self-evolve 是一个「提议—裁决」闭环：**LLM 只负责提出改进，确定性 harness 负责判定
「被采纳 = 真改进」**。它对任何仓库都适用——因为待改进的目标永远变不了方法论，变的
只是「从哪里取得评测信号」。

普适性的根来自一个事实：**任何意图可抽取、可操作化的「更好」，都能被转化为一个可重复的
评测**——有测试就跑测试，有外部事实就核验事实，两者都没有就生成场景与 rubric 让异质判官
评分。（前提：目标意图须可抽取并降维成可判定 rubric；C 的信号有效性仍依赖人审 + 反自欺门。）
**评测信号永远可生成，所以不存在「评不出、因而进化不了」的目标。**

## 为什么没有不可用的目标

旧框定的错误：把 A/B/C 当作**目标的等级**（有测试的是「好目标 A」，纯 markdown 是
「差目标 C，强制人审、跑不出」）。这是把「信号怎么取」误当成「目标能不能进化」。

正确框定：A/B/C 是**同一个目标可叠加使用的三种评测策略**，按「信号可核验强度」排序，
operator 尽量往上取，取不到就向下兜底——**底永远存在**。

论证骨架（三句）：

1. 任何意图可抽取、可操作化的「更好」都能转化为可重复评测。
2. **scenario-eval 是可主动构造的信号发生器**，故 `evaluate` 永远有输出——对任何目标都能
   生成使用场景、从意图写出可打分 rubric、让两个异质模型盲评取一致性。
3. 因此 pipeline 永远能跑完、目标永远可改进。scenario-eval 是 self-evolve 普适性的
   **承重墙**：它把「取不到现成信号」从终点变成起点。

纯 scenario（纯 C）默认走人审落地，**不是因为「目标不可用」，而是因为信号最弱时把最终
判断权交回人**——这是审慎，不是放弃。三个 provider 在 accept 端应当**平权**：能否采纳由统计
强度（e-process + 多闸）决定，而非由「它属于哪一档」结构性歧视。（**落地状态**：此平权是
设计目标；当前代码对纯主观 C 仍保守处理[`coverage=0`、权重偏低]，由 scenario-eval 模块兑现
真 coverage 与平权——见 [`../reference/signal-providers.md`](../reference/signal-providers.md)
的「当前实现 vs scenario-eval 目标」注。）

## 反自欺是不变内核（与评测策略正交）

无论用 A、B 还是 C 取信号，「被采纳 = 真改进」的承诺都靠**同一套反自欺不变量**保证。
它们不绑定任何一种策略，所以三档共用、自举亦共用：

1. **异质核验**——评判者必须与提议者异质：A 用确定性代码（非 LLM）裁决；C 用 Claude×Codex
   双判官且 prompt 不含真值；任何档都「绝不让 LLM 评判自己的产出」。
2. **holdout 隔离**——留一份评测信号（holdout 锚 / 隔离测试真值 / holdout 场景）对
   reflect/propose/patch **物理不可读**，周期抽检，专抓「只在可见信号上变好」的过拟合。
3. **e-process / anytime-valid**——用置信序列累积证据，多轮重评不破坏 type-I，防「多看
   几眼直到偶然变好」。
4. **no-regression 硬门**——任一历史成功项退化即硬 REJECT，凌驾一切统计证据。
5. **背离即升级**——可见信号涨而 holdout 不涨 / judge 与锚背离 / 累计漂移超预算 → 自欺
   多闸触发，强制人审或熔断；释放阀只升人审频率，**绝不降阈自动采纳**。

> 评测策略决定信号的**来源**，反自欺不变量决定信号能不能被**信任**——前者可换，后者不可动。

## 把 self-evolve 套到一个新仓库（operator 心法，5 步）

> 心法：**先想「这个目标的『更好』怎么被独立核验」，按 A→B→C 顺序尽量往强里取信号；
> 取到任意一种就能起跑。**

1. **定义「更好」**——用一两句话写出改进方向（更快 / 更对 / 更清晰 / 覆盖更广）。这句话
   就是后面 rubric 与锚的种子。
2. **找最强可核验信号（A 优先）**——有测试吗？没有，能不能为这句「更好」**生成**一组可
   执行测试（含基线全绿 + 变异校验）？能 → 走 A。
3. **取不到执行判据就找外部事实（B）**——这个「更好」是否系于可独立核验的事实（文档 /
   数据 / 可复现命令的输出）？拆成 ≥ 锚下限的事实锚，留一份 holdout → 走 B。
4. **都没有就生成评测（C，永远成立）**——生成场景 + 从意图写 rubric + 配两个异质判官盲评。
   **到这一步必然能起跑**。纯 C 默认落地走人审。
5. **冻结信号、起跑、看背离**——首次 PROFILE 冻结所用策略（可叠加）；之后只管提议 → 评测
   → 裁决，盯住反自欺信号。沙箱内全自动，**出沙箱一律走人审独立子流程**。

> operator 唯一要做的判断是第 2-4 步「信号取到哪一档」；一旦取到任意一档，方法论与反自欺
> 内核完全复用——这正是普适性的工程含义。
